# Выпускная квалификационная работа

Настоящий репозиторий содержит исходные материалы текста выпускной квалификационной работы (ВКР), а также исходный код разработанного программного модуля и сопутствующий инструментарий для тестирования.

**Тема работы:** Исследование реализации виртуальных криптографических сетевых интерфейсов в ядре OS Linux.

## Обоснование выбора инструментария подготовки текста

Выбор издательской системы LaTeX в качестве основного инструментария для написания ВКР является принципиальным решением. Отказ от использования проприетарных офисных пакетов (в частности, формата `.docx`) продиктован следующими соображениями:

### 1. Соответствие целям работы и принципам информационной безопасности
Ключевой проблематикой данной ВКР является необходимость импортозамещения и минимизация рисков, связанных с использованием технологий, разработанных и стандартизированных на территории недружественных государств. Использование закрытого программного обеспечения (Microsoft Word), разрабатываемого корпорацией, находящейся под юрисдикцией США, противоречило бы концептуальной основе работы.

*   **Риски недекларированных возможностей:** В закрытом ПО невозможно гарантировать отсутствие программных закладок. Доверять обработку материалов работы, посвященной защите информации и государственным стандартам шифрования, проприетарному продукту с закрытым исходным кодом недопустимо с точки зрения информационной гигиены.
*   **Технологический суверенитет:** Форматы на базе открытого текста (`.tex`, `.md`) обеспечивают независимость от конкретного производителя ПО и соответствуют принципам Open Source, лежащим в основе операционной системы Linux, в среде которой выполняется практическая часть исследования.

### 2. Техническая несостоятельность альтернативных редакторов DOCX
В условиях невозможности использования оригинального ПО Microsoft (ввиду причин, описанных в п. 1), единственной альтернативой для редактирования `.docx` являются свободные офисные пакеты (LibreOffice, OnlyOffice и др.). Однако их применение для академических работ большого объема нецелесообразно:

*   **Проблемы совместимости:** Спецификация формата `.docx` является сложной и ориентированной на реализацию в Microsoft Office. Механизмы импорта/экспорта в свободных редакторах не обеспечивают полной совместимости.
*   **Нарушение верстки в объемных документах:** Для сложноструктурированных документов объемом свыше 250 страниц, насыщенных перекрестными ссылками, формулами и плавающими объектами, вероятность некорректного отображения в свободных редакторах критически высока. Это делает работу над текстом диплома в подобных средах ненадежной.

### 3. Требования к инженерной документации
Использование системы LaTeX позволяет избежать описанных выше проблем и соответствует стандартам профессиональной среды. LaTeX является стандартом де-факто для верстки текстов, содержащих сложный математический аппарат, перекрестные ссылки и библиографию, обеспечивая строгое соблюдение ГОСТ при автоматической генерации оформления.

## Требования к программному окружению

Для успешной компиляции документа необходимы следующие компоненты:

1.  **Дистрибутив TeX**: TeX Live (рекомендуется полная установка).
2.  **Компилятор**: `LuaLaTeX` (используется для корректной работы со шрифтами и поддержки Unicode).
3.  **Система автоматизации сборки**: `latexmk`.
4.  **Интерпретатор командной строки**: Bash (или аналог для исполнения скриптов сборки).

## Инструкция по сборке

Сборка документа осуществляется с использованием утилиты `latexmk` с параметром `-shell-escape`, необходимым для работы пакетов подсветки синтаксиса (minted) и выполнения внешних команд.

### Компиляция PDF

Для сборки проекта выполните следующую команду в корневой директории репозитория:

```bash
mkdir -p out && latexmk -shell-escape -pdflua -output-directory=out main.tex
```

В случае успешного выполнения, итоговый файл будет доступен по пути: `out/main.pdf`.

### Очистка временных файлов

Для удаления вспомогательных файлов, созданных в процессе компиляции, используйте команду:

```bash
latexmk -c -output-directory=out
```

## Структура репозитория

Файловая структура проекта организована с целью логического разделения конфигурационных файлов, исходного текста работы, ресурсов и артефактов сборки.

### Корневая директория
*   `main.tex` — Головной файл проекта. Содержит структуру документа и подключение основных модулей.
*   `codebook.toml` — Конфигурационный файл контекста проекта.
*   `tree.txt` — Описание файловой иерархии.

### Директория `cfg/` (Конфигурация)
Содержит файлы преамбулы LaTeX, вынесенные для разгрузки основного файла.
*   `packages.tex` — Подключение используемых пакетов.
*   `settings.tex` — Настройка параметров верстки (шрифты, отступы).
*   `commands.tex` — Определения пользовательских макросов и команд.
*   `scripts.tex` — Скрипты автоматизации внутри LaTeX.

### Директория `src/` (Исходный текст)
Содержит смысловую часть пояснительной записки.
*   `chapters/` — Основной текст работы, разделенный по главам (Введение, Теория, Исследование, Реализация, Заключение).
*   `generic/` — Типовые структурные элементы (Аннотация, Оглавление, Список литературы, Титульный лист).
*   `base.tex`, `body.tex` — Вспомогательные файлы структуры документа.
*   `embeds/` — Внешние вставки:
    *   `code/` — Листинги программного кода (C, Makefile, скрипты).
    *   `tikz/` — Векторная графика и схемы, описанные на языке TikZ.
    *   `console/` — Листинги вывода консоли.

### Директория `res/` (Ресурсы)
Материалы, являющиеся объектом исследования и разработки.
*   `wiregost/` — **Исходный код разработанного модуля ядра**. Содержит реализацию сетевого интерфейса и интеграцию криптографических примитивов.
    *   `gost/` — Реализация алгоритмов ГОСТ (Кузнечик, Стрибог, ЭЦП).
    *   `generated/` — Код взаимодействия с Netlink.
    *   `device.c`, `peer.c` — Основная логика работы виртуального интерфейса.
*   `biblatex/bibliography.bib` — Библиографическая база данных источников.
*   `benchmark/` — Скрипты нагрузочного тестирования и сравнительного анализа производительности.
*   `tests/` — Данные результатов тестирования и скрипты их обработки.

### Директория `out/` (Артефакты)
Директория предназначена для хранения результатов компиляции и временных файлов (`.log`, `.aux`, `.toc`).
